# MySQL

MySQL is an open-source Relational Database Management System (RDBMS) that organizes data into rows and columns. It uses Structured Query Language (SQL) that allows users to retrieve, store, and manage information.

## Data Definition Language (DDL)

These are a category of SQL commands used to modify the structure (schema) of a database.

---

### Create Database

A database is a structured collection of interrelated data stored to serve multiple applications.

#### Syntax

```
CREATE DATABASE [IF NOT EXISTS] database_name;
```

#### Example

```sql
CREATE DATABASE IF NOT EXISTS university_db;
```

---

### Create Table

A table is a structured collection of related data organized into rows and columns.

#### Syntax

```
CREATE TABLE [IF NOT EXISTS] table_name (column_1 datatype_1 [key_1] [constraint_1], ...) [COMMENT = 'desired_comment'];
```

#### Example

```sql
CREATE TABLE IF NOT EXISTS students (
    student_id INT PRIMARY KEY AUTO_INCREMENT,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    email VARCHAR(100) UNIQUE,
    enrollment_date DATE DEFAULT (CURRENT_DATE),
    gpa DECIMAL(3,2) CHECK (gpa >= 0 AND gpa <= 4.0)
) COMMENT = 'Stores student records for university';
```

---

### Create Index

An index is a specialized data structure that improves the speed of data retrieval operations on a table.

#### Syntax

```
CREATE [UNIQUE | FULLTEXT | SPATIAL] INDEX index_name ON table_name (column_list);
```

#### Example

Non-unique index.

```sql
CREATE INDEX idx_student_lastname ON students(last_name);
```

Unique composite index.

```sql
CREATE UNIQUE INDEX idx_unique_name ON students(first_name, last_name);
```

---

### Create View

A view is a virtual table that does not store data physically (except in materialized views) but dynamically fetches results based on a stored query.

#### Syntax

```
CREATE [OR REPLACE] VIEW view_name AS SELECT_statement;
```

#### Example

```sql
CREATE VIEW active_students AS
SELECT student_id, first_name, last_name, gpa
FROM students
WHERE enrollment_date >= '2023-01-01';
```

### Alter Table

Change the name of an existing table.

#### Syntax

```
ALTER TABLE table_name RENAME TO desired_name;
```

#### Example

```sql
ALTER TABLE students RENAME TO learners;
```

Add a new column to an existing table.

#### Syntax

```
ALTER TABLE table_name ADD COLUMN column_name datatype [AFTER desired_column];
```

#### Example

```sql
ALTER TABLE students ADD COLUMN phone VARCHAR(20) AFTER email;
```

Add a new constraint to an existing column.

#### Syntax

```
ALTER TABLE table_name ADD CONSTRAINT constraint_name CHECK (column_name <condition>);
```

#### Example

```sql
ALTER TABLE students ADD CONSTRAINT chk_gpa CHECK (gpa >= 0 AND gpa <= 4.0);
```

Change the name of an existing column.

#### Syntax

```
ALTER TABLE table_name RENAME COLUMN column_name TO desired_name;
```

#### Example

```sql
ALTER TABLE students RENAME COLUMN gpa TO grade_point_average;
```

Change the definition of an existing column.

#### Syntax

```
ALTER TABLE table_name MODIFY COLUMN column_name datatype;
```

#### Example

```sql
ALTER TABLE students MODIFY COLUMN phone VARCHAR(15);
```

Remove an existing constraint in an existing column.

#### Syntax

```
ALTER TABLE table_name DROP CHECK constraint_name;
```

#### Example

```sql
ALTER TABLE students DROP CHECK chk_gpa;
```

Remove an existing column.

#### Syntax

```
ALTER TABLE table_name DROP COLUMN column_name;
```

#### Example

```sql
ALTER TABLE students DROP COLUMN phone;
```

Add an index to an existing column.

#### Syntax

```
ALTER TABLE table_name ADD INDEX index_name (column_name);
```

#### Example

```sql
ALTER TABLE students ADD INDEX idx_email (email);
```

Remove an existing index.

#### Syntax

```
ALTER TABLE table_name DROP INDEX index_name;
```

#### Example

```sql
ALTER TABLE students DROP INDEX idx_email;
```

## Data Manipulation Language (DML)

These are a category of SQL commands used to modify the data of a table in a database.

---

### Insert

Insert a single row into a table by optionally specifying column names or omit to insert all in order.

#### Syntax

```
INSERT INTO table_name [(column1, column2, ...)] VALUES (value1, value2, ...);
```

#### Example

```sql
INSERT INTO students (first_name, last_name, email, gpa)
VALUES ('Michael', 'Jackson', 'michael.jackson@university.edu', 3.8);
```

Insert multiple rows into a table by optionally writing more than one INSERT statement or a single statement for efficiency.

#### Syntax

```
INSERT INTO table_name [(column1, column2, ...)] VALUES (row1_value1, row1_value2, ...), (row2_value1, row2_value2, ...), ...;
```

#### Example

```sql
INSERT INTO students (first_name, last_name, gpa) VALUES
    ('Henry', 'Cavill', 3.9),
    ('Gun', 'Park', 3.4),
    ('Vineeth', 'Chivukula', 3.7);
```

Insert a single row by explicitly assigning values directly to columns.

#### Syntax

```
INSERT INTO table_name SET column1 = value1, column2 = value2, ...;
```

#### Example

```sql
INSERT INTO students SET first_name = 'Clark', last_name = 'Kent', gpa = 4.0;
```

Copy data from another table or query result.

#### Syntax

```
INSERT INTO table_name [(column1, column2, ...)] SELECT_statement;
```

#### Example

```sql
INSERT INTO students_archive (student_id, first_name, last_name)
SELECT student_id, first_name, last_name FROM students WHERE enrollment_date < '2022-01-01';
```

Insert data into a table using default values.

#### Syntax

```
INSERT INTO table_name (column1, column2, column3, ...) VALUES (row1_value1, DEFAULT, row1_value3, ...);
```

#### Example

```sql
INSERT INTO students (first_name, last_name, enrollment_date) VALUES ('Jensen', 'Ackles', DEFAULT);
```

---

### Update

Update existing rows in a table.

#### Syntax

```
UPDATE table_name SET column1 = value1, column2 = value2, ... WHERE condition;
```

#### Example

```sql
UPDATE students SET gpa = 3.85, last_name = 'Jones' WHERE student_id = 101;
```

Update existing rows in a table using an expression.

#### Syntax

```
UPDATE table_name SET column1 = expression1, column2 = expression2, ... WHERE condition;
```

#### Example

```sql
UPDATE students SET gpa = gpa + 0.1 WHERE gpa < 3.5;
```

---

### Delete

Delete existing rows in a table by optionally specifying the condition.

#### Syntax

```
DELETE FROM table_name [WHERE condition];
```

#### Example

```sql
DELETE FROM students WHERE student_id = 999;
```

---

### Replace

Replace a single row in a table.

#### Syntax

```
REPLACE INTO table_name (column1, column2, column3, ...) VALUES (value1, value2, ...);
```

#### Example

```sql
REPLACE INTO students (student_id, first_name, last_name, gpa)
VALUES (101, 'John', 'Wick', 3.9);
```

Replace multiple rows in a table.

#### Syntax

```
REPLACE INTO table_name (column1, column2, column3, ...) SELECT_statement;
```

#### Example

```sql
REPLACE INTO students_archive (student_id, first_name, last_name)
SELECT student_id, first_name, last_name FROM students WHERE gpa > 3.8;
```

## Data Query Language (DQL)

These are a category of SQL commands used to retrieve data from a table.

---

### Select

It is the absolute foundational read-only command used to retrieve data from tables.

#### Syntax

```
SELECT [ALL | DISTINCT] select_list
FROM table_name [AS alias]
[JOIN clauses]
[WHERE condition]
[GROUP BY group_columns]
[HAVING group_condition]
[ORDER BY order_columns [ASC|DESC]]
[LIMIT [offset,] row_count];
```

#### Examples

Select all columns and rows.

```sql
SELECT * FROM students;
```

Select specific columns.

```sql
SELECT first_name, last_name FROM students;
```

Select specific columns with a condition.

```sql
SELECT first_name, last_name, gpa FROM students WHERE gpa > 3.5; -- Can be <, =, >=, <=
```

Select unique values of a column.

```sql
SELECT DISTINCT department FROM courses;
```

Alias specific columns while selecting.

```sql
SELECT first_name AS 'First Name', gpa AS 'Grade Average' FROM students;
```

Select all columns where their values lie within a range.

```sql
SELECT * FROM students WHERE enrollment_date BETWEEN '2023-01-01' AND '2023-12-31';
```

Select all columns where their values match a specific pattern.

```sql
SELECT * FROM students WHERE last_name LIKE 'sm%'; -- Can be %sm (e.g, Lookism), %sm% (e.g, Jasmine)
```

Select all columns where their values match in a defined list.

```sql
SELECT * FROM students WHERE student_id IN (101, 105, 110);
```

Select all columns where their values satisfy all the mentioned conditions.

```sql
SELECT * FROM students WHERE gpa > 3.0 AND enrollment_date > '2024-01-01';
```

Select all columns where their values satisfy either of the mentioned conditions.

```sql
SELECT * FROM students WHERE department = 'CS' OR department = 'Math';
```

Select all columns where their values are not NULL.

```sql
SELECT * FROM students WHERE email IS NOT NULL;
```

Return rows where matching values exist in both tables.

```sql
SELECT s.first_name, s.last_name, m.major_name
FROM students s
INNER JOIN majors m ON s.major_id = m.major_id;
```

Return all rows from the left table and matching rows from the right table (show NULL for unmatched rows).

```sql
SELECT s.first_name, s.last_name, e.course_id
FROM students s
LEFT JOIN enrollments e ON s.student_id = e.student_id;
```

Return all rows from the right table and matching rows from the left table (show NULL for unmatched rows).

```sql
SELECT c.course_name, e.student_id
FROM enrollments e
RIGHT JOIN courses c ON e.course_id = c.course_id;
```

Return every row from the first table paired with every row from the second.

```sql
SELECT s.first_name, c.course_name FROM students s CROSS JOIN courses c;
```

Return rows from a table by matching them against other rows within the same table, treating it as two separate entities using aliases.

```sql
SELECT e1.first_name AS employee_name, e2.first_name AS manager_name
FROM employees e1
LEFT JOIN employees e2 ON e1.manager_id = e2.employee_id;
```

Group rows with identical values in the specified column where all the non-aggregated columns in `SELECT` must be in `GROUP BY`.

```sql
SELECT department, COUNT(*) AS student_count, AVG(gpa) AS avg_gpa
FROM students
GROUP BY department;
```

Group rows with identical values in the specified columns where all the non-aggregated columns in `SELECT` must be in `GROUP BY`.

```sql
SELECT department, enrollment_year, COUNT(*) AS cohort_size
FROM students
GROUP BY department, enrollment_year;
```

Filter groups based on some condition.

```sql
SELECT department, COUNT(*) AS student_count, AVG(gpa) AS avg_gpa
FROM students
WHERE enrollment_date >= '2023-01-01'
GROUP BY department
HAVING AVG(gpa) > 3.2 AND COUNT(*) >= 5;
```

Sort the results in ascending order based on a column.

```sql
SELECT * FROM students ORDER BY first_name; -- Default is ASC
```

Sort the results in descending order based on a column.

```sql
SELECT * FROM students ORDER BY gpa DESC;
```

Sort the results in the desired order based on multiple columns.

```sql
SELECT * FROM students ORDER BY first_name ASC, gpa DESC;
```

Restrict the number of rows.

```sql
SELECT * FROM students LIMIT 7;
```

Restrict the number of rows after skipping a specific offset.

```sql
SELECT * FROM students LIMIT 7 OFFSET 6; -- Can also be written as SELECT * FROM students LIMIT 6, 7;
```

Filter rows based on a single dynamically calculated value from an inner query.

```sql
SELECT first_name, last_name, gpa
FROM students
WHERE gpa > (SELECT AVG(gpa) FROM students); -- Known as a subquery
```

Filter rows by matching against a dynamically generated list of multiple values.

```sql
SELECT first_name, last_name
FROM students
WHERE student_id IN (SELECT student_id FROM enrollments WHERE status = 'active');
```

Treat the results of an inner query as a temporary, dynamically filtered table.

```sql
SELECT MAX(early_students.gpa) AS highest_gpa
FROM (SELECT gpa FROM students WHERE enrollment_date < '2026-01-01') AS early_students;
```

Append a dynamically calculated scalar value as a brand-new column next to every row.

```sql
SELECT first_name, last_name, gpa, (SELECT AVG(gpa) FROM students) AS school_average
FROM students;
```

Combine the result sets of two queries, automatically removing duplicate rows.

```sql
SELECT student_id, first_name, last_name FROM students
UNION
SELECT student_id, first_name, last_name FROM students_archive;
```

Combine the result sets of two queries, preserving all rows including duplicates.

```sql
SELECT student_id, first_name, last_name FROM students
UNION ALL
SELECT student_id, first_name, last_name FROM students_archive;
```

Return only the distinct rows that are present in the result sets of both queries.

```sql
SELECT student_id, first_name, last_name FROM students
INTERSECT
SELECT student_id, first_name, last_name FROM students_archive;
```

Return distinct rows from the first query that do not exist in the result set of the second query.

```sql
SELECT student_id, first_name, last_name FROM students
EXCEPT
SELECT student_id, first_name, last_name FROM students_archive;
```

Evaluate a list of sequential conditions and return a specific value for the first true condition encountered (Searched `CASE`).

```sql
SELECT first_name, gpa,
       CASE WHEN gpa >= 3.5 THEN 'Honors List'
            WHEN gpa >= 2.0 THEN 'Good Standing'
            ELSE 'Academic Probation'
       END AS academic_status
FROM students;
```

### Execution Order

The chronological sequence in which the database engine processes each clause and returns data:

```
FROM | JOIN -> WHERE -> GROUP BY -> HAVING -> SELECT -> DISTINCT -> ORDER BY -> LIMIT | OFFSET
```

#### Example

```sql
SELECT c.department AS dept_name, AVG(s.gpa) AS average_gpa  -- (5) Evaluates columns & aliases
FROM students s                                               -- (1) Finds the primary data source
INNER JOIN enrollments e ON s.student_id = e.student_id       -- (1) Merges enrollment data
INNER JOIN courses c ON e.course_id = c.course_id             -- (1) Merges course details
WHERE e.status = 'active' AND c.course_level > 100            -- (2) Filters base rows before grouping
GROUP BY c.department                                         -- (3) Groups remaining rows into buckets
HAVING AVG(s.gpa) >= 3.0                                      -- (4) Filters aggregated group buckets
ORDER BY average_gpa DESC                                     -- (6) Sorts the final output using the alias
LIMIT 2;                                                      -- (7) Restricts total rows returned
```

### Aggregation Functions

These are functions that perform calculations on multiple rows of a single column to return a single summarized value.

#### Syntax

```
SELECT AGGREGATE_FUNCTION([DISTINCT] column_name | *) FROM table_name;
```

#### Examples

Count the total number of records in a table including rows with NULL values.

```sql
SELECT COUNT(*) FROM students;
```

Count the total number of non-null values in a specific column.

```sql
SELECT COUNT(email) FROM students;
```

Count the total number of unique values in a column by eliminating duplicates.

```sql
SELECT COUNT(DISTINCT department) FROM courses;
```

Calculate the total arithmetic sum of all numeric values in a column.

```sql
SELECT SUM(tuition_paid) FROM invoices;
```

Calculate the mathematical mean of all numeric values in a column, automatically ignoring NULL values.

```sql
SELECT AVG(gpa) FROM students;
```

Find the highest numeric value, latest date, or last alphabetical string in a column.

```sql
SELECT MAX(gpa) FROM students;
```

Find the lowest numeric value, earliest date, or first alphabetical string in a column.

```sql
SELECT MIN(enrollment_date) FROM students;
```

### String Functions

These are scalar functions that perform operations on text data to manipulate, format, or extract characters from strings.

#### Syntax

```
SELECT STRING_FUNCTION(string_column | 'text_literal' [, arguments]) FROM table_name;
```

#### Examples

Convert all characters in a string to uppercase.

```sql
SELECT UPPER(first_name) FROM students;
```

Convert all characters in a string to lowercase.

```sql
SELECT LOWER(first_name) FROM students;
```

Calculate the total number of characters in a string, including spaces and punctuation in bytes.

```sql
SELECT LENGTH(last_name) FROM students;
```

Return the character length of a string specifically counting multibyte characters as a single character.

```sql
SELECT CHAR_LENGTH(last_name) FROM students;
```

Combine two or more strings together into a single text output.

```sql
SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM students;
```

Combine two or more strings together with a separator into a single text output.

```sql
SELECT CONCAT_WS(', ', first_name, last_name) AS full_name_ws FROM students;
```

Extract a specific portion of a text string based on a starting position and a defined length.

```sql
SELECT SUBSTRING(email, 1, 5) FROM students; -- Extracts the first 5 characters
```

Replace all occurrences of a specified substring within a text string with a new text value.

```sql
SELECT REPLACE(email, 'university.edu', 'alumni.edu') FROM students;
```

Remove any leading and trailing space characters from a text string.

```sql
SELECT TRIM('  Patrick Bateman  ') AS cleaned_name FROM students;
```

Remove any leading space characters from a text string.

```sql
SELECT LTRIM('  Ned Luke') AS cleaned_name FROM students;
```

Remove any trailing space characters from a text string.

```sql
SELECT RTRIM('Ada Wong  ') AS cleaned_name FROM students;
```

Find the first occurrence position of a substring inside a target string (returns 0 if not found).

```sql
SELECT INSTR(email, '@') AS at_symbol_position FROM students;
```

Pad the left side of a string with a specific character string until it reaches a desired total length.

```sql
SELECT LPAD(student_id, 8, '0') AS padded_id FROM students; -- Turns '123' into '00000123'
```

Pad the right side of a string with a specific character string until it reaches a desired total length.

```sql
SELECT RPAD(department, 5, '.') AS formatted_dept FROM courses; -- Turns 'CS' into 'CS...'
```

Reverse the order of all characters within a string.

```sql
SELECT REVERSE(first_name) FROM students;
```

### Date and Time Functions

These are scalar functions used to retrieve, manipulate, format, or perform calculations on date, time, and timestamp values.

#### Syntax

```
SELECT DATE_FUNCTION([date_column | 'date_literal' [, arguments]]) FROM table_name;
```

#### Examples

Return the current system date and time as a timestamp value.

```sql
SELECT NOW() AS current_timestamp_now; -- Format: YYYY-MM-DD HH:MM:SS
```

Return the current system date value without the time component.

```sql
SELECT CURDATE() AS current_date_today; -- Format: YYYY-MM-DD
```

Extract the four-digit year from a date value.

```sql
SELECT YEAR(enrollment_date) FROM students;
```

Extract the month number (1 to 12) from a date value.

```sql
SELECT MONTH(enrollment_date) FROM students;
```

Extract the day of the month (1 to 31) from a date value.

```sql
SELECT DAY(enrollment_date) FROM students;
```

Add a specified time interval value to a date or _datetime_ value.

```sql
SELECT DATE_ADD(enrollment_date, INTERVAL 4 YEAR) AS graduation_date FROM students;
```

Subtract a specified time interval value from a date or _datetime_ value.

```sql
SELECT DATE_SUB(enrollment_date, INTERVAL 6 MONTH) AS orientation_date FROM students;
```

Calculate the total number of days between two distinct date values (expressed as Date1 - Date2).

```sql
SELECT DATEDIFF('2026-06-06', enrollment_date) AS days_enrolled FROM students;
```

Format a date value into a customized string format based on specified format specifier codes.

```sql
SELECT DATE_FORMAT(NOW(), '%Y-%m-%d %H:%i:%s');
```

Extract only the time component from a combined _datetime_ or timestamp expression.

```sql
SELECT TIME(NOW()) AS current_time_only; -- Format: HH:MM:SS
```

### Numeric Functions

These are scalar functions used to perform mathematical calculations, rounding operations, or algebraic manipulations on numeric data.

#### Syntax

```
SELECT NUMERIC_FUNCTION(numeric_column | number_literal [, arguments]) FROM table_name;
```

#### Examples

Return the absolute, positive value of a number by stripping its negative sign.

```sql
SELECT ABS(-3.5) AS absolute_value FROM students; -- Returns 3.5
```

Round a numeric value to a specified number of decimal places.

```sql
SELECT ROUND(gpa, 1) FROM students; -- e.g., 3.85 becomes 3.9
```

Truncate a numeric value down to a specified number of decimal places without rounding.

```sql
SELECT TRUNCATE(gpa, 1) FROM students; -- e.g., 3.85 becomes 3.8
```

Round a numeric value up to the next nearest whole integer.

```sql
SELECT CEIL(gpa) FROM students; -- e.g., 3.1 becomes 4.0
```

Round a numeric value down to the previous nearest whole integer.

```sql
SELECT FLOOR(gpa) FROM students; -- e.g., 3.9 becomes 3.0
```

Calculate the remainder of a division operation between two numbers.

```sql
SELECT MOD(student_id, 2) AS is_odd FROM students; -- Returns 0 for even IDs, 1 for odd IDs
```

Raise a base number to the exponential power of a specified exponent value.

```sql
SELECT POW(gpa, 2) FROM students; -- Calculates gpa squared
```

Calculate the non-negative square root of a numeric value.

```sql
SELECT SQRT(9) AS square_root FROM students; -- Returns 3.0
```

Return the mathematical constant Pi (π).

```sql
SELECT PI() AS pi_value FROM students; -- Returns 3.141593
```

### NULL Functions

These are conditional scalar functions used to handle, replace, or evaluate missing or unassigned values (NULL) within your dataset.

#### Syntax

```
SELECT NULL_FUNCTION(expression1 [, expression2, ...]) FROM table_name;
```

#### Examples

Return the second argument if the first argument evaluates to a NULL value.

```sql
SELECT IFNULL(email, 'no-email@university.edu') AS contact_email FROM students;
```

Return the very first non-null value from a specified list of arguments evaluated from left to right.

```sql
SELECT COALESCE(phone, email, 'No Contact Info Available') AS primary_contact FROM students;
```

Compare two arguments and return NULL if they are equal; otherwise, return the first argument.

```sql
SELECT NULLIF(department, 'Undeclared') AS verified_dept FROM students; -- Returns NULL if department is 'Undeclared'
```

Evaluate a condition and return a specified value if it is true, and an alternative value if it is false.

```sql
SELECT IF(gpa >= 2.0, 'Good Standing', 'Academic Probation') AS student_status FROM students;
```

### Window Functions

These are advanced functions that perform calculations across a specific set of table rows (a "window") that are relationally related to the current row, preserving the individual identity of each row in the final output.

#### Syntax

```
SELECT WINDOW_FUNCTION(expression) OVER (
    [PARTITION BY partition_columns]
    [ORDER BY sort_columns]
    [ROWS|RANGE frame_spec]
) FROM table_name;
```

#### Examples

Assign a unique, sequential integer to each row starting at 1 within every department partition.

```sql
SELECT department, last_name,
       ROW_NUMBER() OVER (PARTITION BY department ORDER BY gpa DESC) AS rank_in_dept
FROM students;
```

Assign a rank to each row within a partition where tied values receive the same rank, leaving gaps in the sequence afterward.

```sql
SELECT first_name, gpa,
       RANK() OVER (ORDER BY gpa DESC) AS national_rank
FROM students; -- Tied GPAs get the same rank (e.g., 1, 2, 2, 4)
```

Assign a rank to each row within a partition where tied values receive the same rank, preserving a continuous numerical sequence without gaps.

```sql
SELECT first_name, gpa,
       DENSE_RANK() OVER (ORDER BY gpa DESC) AS continuous_rank
FROM students; -- Tied GPAs get the same rank with no gaps (e.g., 1, 2, 2, 3)
```

Calculate a running or partition-wide aggregate total without compressing individual table rows.

```sql
SELECT department, first_name, tuition_paid,
       SUM(tuition_paid) OVER (PARTITION BY department) AS total_dept_revenue
FROM students;
```

Calculate a cumulative running total from the start of the partition up to the current row.

```sql
SELECT department, enrollment_date, tuition_paid,
       SUM(tuition_paid) OVER (
           PARTITION BY department
           ORDER BY enrollment_date ASC
           ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
       ) AS rolling_total
FROM students;
```

Retrieve a value from a previous row within the same partition based on a designated physical offset.

```sql
SELECT student_id, enrollment_date,
       LAG(enrollment_date, 1) OVER (ORDER BY enrollment_date ASC) AS previous_student_date
FROM students;
```

Calculate a rolling average using a physical frame of the current row and the immediate two rows preceding it.

```sql
SELECT enrollment_date, gpa,
       AVG(gpa) OVER (
           ORDER BY enrollment_date ASC
           ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
       ) AS moving_avg_3_students
FROM students;
```

Retrieve a value from a succeeding row within the same partition based on a designated physical offset.

```sql
SELECT student_id, enrollment_date,
       LEAD(enrollment_date, 1) OVER (ORDER BY enrollment_date ASC) AS next_student_date
FROM students;
```

Calculate a moving aggregate using values based on the data value itself (logical range) rather than physical row positions.

```sql
SELECT student_id, gpa,
       COUNT(*) OVER (
           ORDER BY gpa
           RANGE BETWEEN 0.1 PRECEDING AND 0.1 FOLLOWING
       ) AS peers_within_point_one_gpa
FROM students;
```

Divide the ordered rows within a partition into a specified number of roughly equal, ranked groups (tiles).

```sql
SELECT first_name, gpa,
       NTILE(4) OVER (ORDER BY gpa DESC) AS gpa_quartile
FROM students; -- Divides the student body into 4 performance quartiles (1 to 4)
```

### Regular expression (Regex) Functions

These are advanced pattern-matching functions used to search, extract, verify, and manipulate complex text strings that standard `LIKE` wildcards cannot handle. It is **recommended** to refer to the regex cheat sheet before diving into the examples.

#### Syntax

```
SELECT REGEXP_FUNCTION(string_column, 'regex_pattern' [, arguments]) FROM table_name;
```

#### Examples

Find records where a text column matches a specific pattern using the classic regex operator.

```sql
SELECT first_name, email
FROM students
WHERE email REGEXP '^[A-Za-z0-9._%+-]+@university\\.edu$'; -- Finds only official university emails
```

Evaluate whether a string matches a regular expression pattern, returning 1 if it matches and 0 if it does not.

```sql
SELECT course_code, title
FROM courses
WHERE REGEXP_LIKE(course_code, '^CS-[34][0-9]{2}$'); -- Matches 300 and 400 level Computer Science courses (e.g., CS-301)
```

Return the starting position index of the first substring matching the regular expression pattern (return 0 if no match is found).

```sql
SELECT phone, REGEXP_INSTR(phone, '[0-9]{4}$') AS last_four_digits_start
FROM students; -- Returns the starting character position of the final 4-digit block
```

Extract the exact substring that matches a regular expression pattern from a target text string.

```sql
SELECT email, REGEXP_SUBSTR(email, '@[A-Za-z0-9.-]+') AS email_domain
FROM students; -- Extracts everything from the '@' symbol to the end of the string
```

Search a string for a regular expression pattern and replace all matching occurrences with a new specified replacement string.

```sql
SELECT phone, REGEXP_REPLACE(phone, '[^0-9]', '') AS clean_numeric_phone
FROM students; -- Strips away all dashes, spaces, and parentheses, leaving only raw numbers
```

### Common Table Expressions (CTEs)

A Common Table Expression is a temporary named result set that you can reference within a subsequent `SELECT`, `INSERT`, `UPDATE`, or `DELETE` statement. They act as virtual tables that exist only for the duration of that single query execution, making complex queries far more readable than deeply nested subqueries.

#### Syntax

```
WITH cte_name [(column_list)] AS (
    SELECT_statement_defining_the_cte
)
SELECT_statement_using_the_cte;
```

#### Examples

Define a single temporary result set to pre-calculate high-performing students, then filter the virtual dataset in the main query.

```sql
WITH honor_roll AS (
    SELECT student_id, first_name, last_name, gpa
    FROM students
    WHERE gpa >= 3.5
)
SELECT first_name, last_name, gpa
FROM honor_roll
WHERE gpa > 3.9;
```

Define multiple, comma-separated CTEs where the second CTE references the first one to create a modular data processing pipeline.

```sql
WITH dept_averages AS (
    SELECT department, AVG(gpa) AS avg_gpa
    FROM students
    GROUP BY department
),
top_departments AS (
    SELECT department, avg_gpa
    FROM dept_averages
    WHERE avg_gpa >= 3.0
)
SELECT d.department, d.avg_gpa, c.title
FROM top_departments d
INNER JOIN courses c ON d.department = c.department;
```

### Recursive CTEs

A specialized variation of a CTE that references itself. It is used to traverse hierarchical, graph-based, or tree-structured data by looping through rows until a termination condition is met.

#### Syntax

```
WITH RECURSIVE cte_name AS (
    -- Anchor Member: Establishes the base starting row(s)
    SELECT_statement1

    UNION [ALL]

    -- Recursive Member: References cte_name to loop through next levels
    SELECT_statement2 WHERE termination_condition
)
SELECT * FROM cte_name;
```

#### Examples

Traverse a hierarchical course prerequisite chain to find all sequential courses required to unlock an advanced 400-level course.

```sql
WITH RECURSIVE course_path AS (
    -- Anchor Member: Start with the advanced target course
    SELECT course_id, title, prerequisite_id
    FROM courses
    WHERE title = 'Advanced Quantum Physics'

    UNION ALL

    -- Recursive Member: Join the CTE back to the base table to find the next prerequisite row
    SELECT c.course_id, c.title, c.prerequisite_id
    FROM courses c
    INNER JOIN course_path cp ON c.course_id = cp.prerequisite_id
)
SELECT * FROM course_path;
```
